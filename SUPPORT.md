# Support

markpad is an early open-source project. The best support path depends on what you need.

## Bug Reports

Open a bug report issue when something is broken. Include reproduction steps, expected behavior, actual behavior, operating system, and app version or commit SHA.

## Feature Ideas

Open a feature/spec request issue before implementation. Include the user problem, acceptance criteria, and any design constraints.

## Questions

Use an issue with the `question` label for now. If project discussion grows, support questions may move to GitHub Discussions.

## Security

Do not open a public issue for suspected vulnerabilities. Follow `SECURITY.md` instead.
